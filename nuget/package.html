Package

Package

Compilare i campi presenti in Assemby Information della vostra libreria. Queste informazioni inserite verranno visualizzate in NuGet.

Nuspec

E' necessario creare il file .nuspec che serve a definire le proprietà del pacchetto. Posizionarsi nella cartella del file .csproj ed eseguire il comando

nuget spec NomeProgetto.csproj

verrà generato un file .nuspec simile al seguente con le info del pacchetto che verrà generato.

<?xml version="1.0"?>
<package >
  <metadata>
    <id>$id$</id>
    <version>$version$</version>
    <title>$title$</title>
    <authors>$author$</authors>
    <owners>$author$</owners>
    <projectUrl>https://www.mstchannel.com</projectUrl>
    <iconUrl>https://uat.mstchannel.com/assets/icons/icon-72x72.png</iconUrl>
    <requireLicenseAcceptance>false</requireLicenseAcceptance>
    <description>$description$</description>
    <releaseNotes>Prima versione appender per ELK Stack.</releaseNotes>
    <copyright>Copyright 2019</copyright>
    <tags>Log4Net Elasticsearch Log</tags>
    <dependencies>
        <dependency id="log4net" version="2.0.8" />
        <dependency id="Newtonsoft.Json" version="12.0.1" />
    </dependencies>
  </metadata>
</package>

Le info tra $ verranno automaticamente compilate partendo dall'assemblyinfo le altre dovranno essere inserite manualmente.

Creazione pacchetto nuget

Dopo aver effettuato la build della vostra libreria in Release posizionarsi nella cartella del file .csproj ed eseguire il comando

nuget pack NomeProgetto.csproj -Prop Configuration=Release

verrà creato il file .nupkg

Pubblicazione pacchetto nuget

Dopo aver creato il file .nupkg è necessario fare l'upload nel feed eseguendo il comando

nuget add NomePacchetto.nupkg -source \\[ipmacchina]\dati\Reparto_IT\Nuget
